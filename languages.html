Agda

Beginner

 

 

I’m still at the beginning of my journey with Agda, but I’m really enjoying where it’s taking me. Agda gives me two things that I’ve been wanting in a programming language: interactive programming (or theorem proving, depending on your perspective) and formal verification at the code level.

I like that Agda lets me iteratively explore the possible inhabitants of a type signature and that my code won’t compile unless it is not only well-formed but also correct. The only thing that I don’t like about Agda is that proofs must be written explicitly; it sometimes junks up what would otherwise be easily readable and obviously provable code.

C#

 

 

Advanced

Since around 2014, or whenever they removed Boo, C# has been the de facto scripting language of the Unity game engine. I’ve used Unity a lot personally, professionally, and academically. This means that I can never really get away from C#. I don’t mess with anything .NET unless I have to; unfortunely, I often have to go outside the limited Mono implementation that comes with Unity to create those tasty machine learning features in some of my games.

It’s hard to explain to people that C# is a largely imperative, object-oriented language with Java-like syntax and garbage collection without it sounding like a nightmare. The Unity profiler tells me that the garbage collector is almost always doing most of the work, but to my surprise it never seems to collect C# itself. I like LINQ because I love to write in a functional style and I hate when my users have excess cycles.

Chinese (Mandarin)

 

Intermediate

 

In the summer of 2021, I started learning Mandarin as a joke. Now I regularly listen to Chinese podcasts and watch Chinese cartoons, as the boomers say. I read way better than I write; I can write 我有一隻小狗, but I can’t write 𰻞. I can speak a little too, but the only time I’ve really practiced is when I visited my sister in Taiwan.

Beyond its phonology, I like that the grammar of Chinese is pretty much entirely syntactic, rather than morphological. I find it fascinating how new meanings are suggested explicitly using smaller semantic units (i.e., Chinese characters). A personal favorite is the word for computer: 電腦, which can be parsed as “electric brain.” 請跟我說一點中文!!

Haskell

 

Intermediate

 

I like using Haskell because it lets me write code that looks like how I think about it. With its explicit type system and necessary purity, good style is basically a language feature. It’s easy to favor composition over inheritance if you just don’t have any inheritance. I find that I can write modules that just do what I need them to do and that I never have to refactor.

Although I can read a good deal of Haskell, I seldom have a chance to write any. It’s far too tempting to just hack something together in Python. I did, however, use a Haskell package called Hakyll to generate this site because I’d much rather read and modify others’ Haskell code over code written in any other language. The code just means what it says; I don’t even have to write a show instance to know what’s going on.
